Quick Start

Six commands. One complete contribution.

# Find an issue to work on
/contribute discover

# Analyze the repo and plan your approach
/contribute analyze https://github.com/owner/repo/issues/123

# Write the code
/contribute work

# Validate with 5-stage testing (must score 85%+)
/contribute test

# Push and open PR
/contribute submit

# Monitor feedback and iterate
/contribute review

Or just type /contribute with no arguments -- it auto-detects the right phase from your current context.

Core Workflow

Six phases that take you from zero to merged PR.

1

Discover

/contribute discover

Search GitHub for issues matching your skills. Parallel searches with quality scoring. Verifies issues aren't already taken.

2

Analyze

/contribute analyze URL

Clone repo, read CONTRIBUTING.md, trace code paths, produce a structured brief with files to modify and recommended approach.

3

Work

/contribute work

Implement the change. TDD-first. Match upstream style exactly. Minimal, focused changes. Three operating modes.

4

Test

/contribute test

5-stage validation: upstream tests, static analysis, security audit, functional verification, AI deep review. 85% gate.

5

Submit

/contribute submit

Rebase on upstream, push to fork, draft PR following project conventions, open after your approval.

6

Review

/contribute review

Check CI status, summarize maintainer feedback, iterate on requested changes, draft professional responses.

Three Operating Modes

Choose how much the AI does.

do

AI writes everything -- code, tests, formatting, validation.

You: review diffs, approve writes

guide

AI explains each step, provides snippets and commands.

You: execute the commands, write the code

adaptive

AI handles boilerplate, scaffolding, mechanical work.

You: logic, design decisions, algorithms

Safety Rules

Six non-negotiable rules govern every action.

  • 1
    Read is free, write requires permission
    Every fork, branch, commit, push, and comment requires your explicit approval.
  • 2
    No AI attribution
    Every commit appears as sole-authored by you. Zero AI trailers. Zero exceptions.
  • 3
    Three operating modes
    You choose how much autonomy the AI gets: do, guide, or adaptive.
  • 4
    Respect upstream conventions
    Matches the project's style exactly. Tabs, naming, commit format -- all mirrored.
  • 5
    Opus only
    Both agents run on Claude Opus for maximum quality. No shortcuts.
  • 6
    Verify issue is not taken
    Checks assignees, open PRs, comment claims, and linked PRs before recommending any issue.
